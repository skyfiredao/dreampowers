# Dreampowers

**[English](README.md)**

一套完整的 AI 技能集，专为**中文网络小说写作**设计，基于 [oh-my-openagent](https://github.com/code-yeongyu/oh-my-openagent) 框架构建。14 个专业技能覆盖小说创作的每个阶段，从故事构思到成稿交付，核心创新在于**防止世界观信息倾泻**、**读者视角体验测试**和 **AI味检测消除**。

## 功能

Dreampowers 为 AI 辅助中文小说写作提供了一套有纪律的、技能驱动的工作流。它不会把所有世界观设定一股脑塞进第一章，而是通过量化规则和多阶段审查，强制执行渐进式揭示。

- **14 个专业技能**，覆盖小说写作全生命周期
- **三阶段审查**，每章必过（情节、揭示检查、文笔）
- **外部审阅闭环**：三阶段审查后进入读者审阅 + 一致性审阅，最多循环 3 次，仍有问题则标记 TBD 交作者人工审阅
- **六条铁律**，零容忍防止信息倾泻
- **伏笔场记**，跨全稿追踪埋设与回收
- **场景类型导演**，含动作、情感、对话三种子模式
- **四种连续写作模式**：逐章确认、连写N章、写完一卷暂停、全自动
- **叙事时间线手法**：顺叙/倒叙/插叙/补叙，附 POV 选择与切换规则
- **描写方法论**：镜头语言体系（由远及近、粗细结合、动静结合），融入场景导演
- **外部内容导入**：接入已有世界观设定、角色卡、大纲及手写章节
- **中途改纲**：7 步冻结-修订工作流，支持创作中途的方向调整
- **读者视角测试**：冷读模拟，评估翻页欲、认知负荷、共情连接、节奏感受
- **AI味检测**：量化模式匹配，消除套路化 AI 写作习惯
- **写作风格定义**：七维问卷（叙事距离、句法节奏、感官密度、词汇层级、比喻策略、情感表达、对话比重），内置 9 类题材 54+ 参考作者
- **选装成人内容**：成人场景写作，含感官完整性与叙事真实框架、两级制（软性/硬核）、用户自定义 `adult.md` 偏好，通过 `--all` 参数单独安装
- **三层隔离**，与编程技能互不干扰（入口切换 + `dp-` 前缀 + 目录隔离）
- **基于 Git 的版本管理**，管理稿件版本

## 技能一览

### 设定技能（准备阶段）

| 技能 | 用途 |
|---|---|
| `dp-using-dreampowers` | 入口技能，负责模式切换、技能路由、工作流概览 |
| `dp-tool-research` | 故事前提脑暴、一句话故事概要公式、故事命名、平台对比、互联网辅助世界观调研、作者调优 |
| `dp-set-style` | 作品级写作风格定义，七维问卷（叙事距离、句法节奏、感官密度、词汇层级、比喻策略、情感表达、对话比重），风格参考库（9 类题材 54+ 作者），生成 style.md 风格档案 |
| `dp-set-concept` | 世界观/角色设定，含冰山标注（50%+ 沉在水下），文件级概念隔离，故事级时间线构建，外部素材导入 |
| `dp-set-outline` | 卷/章结构、4 种模型、6 条铁律与概念预算、伏笔场记联动、主题编织、叙事时间线手法、POV 规则、中途改纲 |

### 章节技能（写作阶段）

| 技能 | 用途 |
|---|---|
| `dp-chapter-draft` | 章节写作，含草稿预审、三阶段审查、4 种连续写作模式、时间线执行指引、手写章节接入、时代准确性写作（时代物品与视角校验）、大纲框架纪律 |
| `dp-chapter-summary` | 章节摘要生成（纯文本≤150字，无格式，唯一输入为 `docs/dreampowers/release/chapter-NNN.md`），用于跨章连续性 |
| `dp-chapter-direct` | 场景类型导演（动作/情感/对话子模式）+ 叙事节奏控制、张弛法则、镜头语言描写方法论 |
| `dp-chapter-adult` | **选装**：成人场景写作，含感官完整性与叙事真实框架、两级制（软性/硬核）、用户自定义 `adult.md` 偏好 |

### 工具技能

| 技能 | 用途 |
|---|---|
| `dp-character-style` | 角色风格档案、遮名测试、对话规则、潜台词技法 |
| `dp-tool-version` | 基于 Git 的版本管理：结构化提交、回滚、差异对比 |

### 审查技能

| 技能 | 用途 |
|---|---|
| `dp-review-reader` | 读者视角体验测试：翻页欲、认知负荷、共情验证、节奏感受 |
| `dp-review-consistency` | 跨章连续性校验（9 个维度）+ 修订建议与去AI味 |
| `dp-review-final-report` | 全书终检报告：全书完成后，对照世界观设定、角色、大纲、时间线，扫描所有已发布章节的全局一致性（人工可选触发） |

## 核心创新

### 设定揭示节奏控制 (dp-set-outline)

六条铁律，零容忍执行：

1. 先制造悬念，再给出答案
2. 先呈现感官体验，再解释规则体系
3. 先深化已有概念，再引入新概念
4. 第一章最多引入 3 个新概念（可以为 0），后续每章最多 2 个（可以为 0）
5. 所有世界观信息只能通过角色行动/对话传达
6. 只揭示当前冲突所需的信息

### 伏笔场记

`docs/dreampowers/tracking/` 目录下以 `thread-` 为前缀的独立 Markdown 文件，追踪所有伏笔线索：
- 伏笔回收由大纲规划决定，一致性检查确认
- Claremont 系数（CC = 已埋设 - 已回收），CC > 2 时触发预警
- 逾期线索检测

### 概念隔离 (dp-set-concept)

文件级物理隔离，防止 AI 看到未来的世界观概念或角色状态：

- **一概念一文件**，存放于 `docs/dreampowers/set/concept/`
- **一角色一文件或一目录**，存放于 `docs/dreampowers/set/character/`
- **章节文件夹**（`docs/dreampowers/chapters/chapter-NNN/`），完全自包含的写作单元
- **章节文件夹内包含**：概念/角色/伏笔的符号链接 + `spec.md`（七节章节 spec：大纲阶段框架§1-5 + 草稿预审评估§6 + 自包含写作蓝图§7）
- **铁律文件**从 `docs/dreampowers/tracking/iron-rules.md` 符号链接到各章节文件夹
- **前序章节摘要**符号链接（全部前序章节）以保持上下文连续性
- **草稿预审**读取章节文件夹全部材料，浓缩写入 `spec.md` §6-7，用户确认后**写作阶段只读 `spec.md`**
- **串行章节写作**：必须逐章顺序写作，禁止并行起草

### 三阶段章节审查

每一章都必须依次通过三个审查阶段：
1. **情节审查**：冲突推进、角色一致性、大纲符合度
2. **揭示检查**：零容忍信息倾泻检测，对照六条铁律逐项排查
3. **文笔审查**：风格一致性、对话声音、感官细节、行文节奏

### 外部审阅闭环

三阶段审查通过后，每章进入外部审阅闭环：
1. **读者审阅**（dp-review-reader）：冷读者视角的四维度体验测试
2. **一致性审阅**（dp-review-consistency）：九维度连续性检查 + 修订建议与AI味检测 + 对照 style.md 的写作风格检查
3. **合并修改并循环**：汇总两份报告的问题统一修改，然后重新进入闭环（最多 3 次）
4. **TBD 兜底**：3 次后仍有问题，终稿保存为 `chapter-NNN-TBD.md`，交给作者人工审阅

### 叙事技法

跨技能统一集成，确保技法一致性：

- **时间线手法**（dp-set-outline、dp-chapter-draft）：顺叙、倒叙、插叙、补叙，附决策树选择指引，以及章节层面的执行要点
- **POV 规则**（dp-set-outline）：第一人称/有限第三/全知第三对比，5 条硬规则（仅在场景边界切换、情感锚点、禁止信息走私、场景内一致、过渡标记）
- **描写方法论**（dp-chapter-direct）：镜头语言体系，与三种场景子模式深度融合

### 连续写作与中途改纲

- **四种写作模式**：逐章确认（默认）、连写N章、写完一卷暂停、全自动（含质量关卡）
- **中途改纲**：7 步冻结-修订工作流

### 读者视角测试 (dp-review-reader)

模拟"冷读者"，不参考大纲、世界观设定、角色卡，仅凭已发布章节的累积信息进行体验评估。四个测试维度：

- **翻页欲**：读者想不想继续读？（1-5 分，2 分及以下标红）
- **认知负荷**：读者是否困惑或信息过载？（反向量表，500 字内超过 3 个陌生元素 = 超载）
- **共情验证**："为什么我应该在乎？"测试，检验角色动机清晰度
- **节奏感受**："想跳过这段"（拖沓）vs "等等发生了什么"（过快）

### AI味检测 (dp-review-consistency)

跨六个层面的量化模式匹配，消除 AI 写作习惯：

- **词汇层**：比喻词泛滥（仿佛/宛如每千字不超过 2 次）、情感标签（感到/觉得不超过 1 次）、程度副词堆砌
- **句式层**：排比三连反射、句长标准差过低（低于 3 字 = 机械节奏）、主语重复
- **结构层**：每场景固定三段式、公式化转场、每段结尾必升华
- **修辞层**：通感滥用、比喻密度过高、万物拟人、伪古旧感（回忆滤镜，对过去时代加当代怀旧视角）、时代物品穿帮（不属于故事年代的器物）
- **格式层**：连字符/长破折号入侵（零容忍）、Emoji 表情符号（零容忍）、Markdown 星号标记（零容忍）、HTML 实体字符（零容忍）
- **语域层**：专业术语密度、定义式句型、被动/无主语结构、枚举式行文、过度显化因果链

## 前置条件

- [Git](https://git-scm.com/)（用于 `dp-tool-version` 的稿件版本管理）
- 已安装 [OpenCode](https://opencode.ai) 及 [oh-my-openagent](https://github.com/code-yeongyu/oh-my-openagent)

## 安装

```bash
# 克隆或复制 skill_dreampowers 目录
cd skill_dreampowers

# 安装默认技能（13 个，跳过选装技能）
chmod +x install.sh
./install.sh

# 或安装全部技能（含选装，如 dp-chapter-adult）
./install.sh --all
```

安装脚本会为每个 `dp-*` 技能目录创建指向 `~/.config/opencode/skills/` 的符号链接。选装技能（如 `dp-chapter-adult`）默认跳过，使用 `--all` 或手动创建符号链接安装。

## 卸载

```bash
./uninstall.sh
```

移除 `~/.config/opencode/skills/` 下所有 `dp-*` 符号链接。

## 使用方法

### 开始使用

在 OpenCode 会话中切换到创作模式：

```
Use skill dp-using-dreampowers
```

这将激活 Dreampowers 路由表，AI 将对所有创作任务使用 `dp-*` 技能。

**从零开始写一部新小说**，按完整流程走：

```
我想写一个仙侠小说
```

AI 加载 `dp-tool-research`，引导你完成故事前提脑暴（题材、一句话故事概要、平台适配）。确认前提后自动进入下一阶段：

```
dp-tool-research --> dp-set-style --> dp-set-concept --> dp-set-outline --> dp-chapter-draft（循环）
```

每个阶段产出制品并自动衔接下一步。你不需要手动调用技能，路由表自动处理过渡。

### 导入已有素材

如果你已经有世界观文档、角色卡、大纲或已写好的章节，使用导入入口：

```
我有一份已经写好的世界观设定，帮我导入
```

AI 加载 `dp-set-concept`，识别内容类型，进行对应处理：

| 你提供的内容 | 处理方式 |
|---|---|
| 世界观设定文档 | 冰山标注（50%+ 标记为"水下"），存入 `docs/dreampowers/set/world/` |
| 角色卡 | 提取角色风格档案，存入 `docs/dreampowers/set/character/` |
| 故事大纲 | 经 `dp-set-outline` 做结构校验、伏笔规划 |
| 已写章节 | 元数据注入、伏笔回溯、连续性校验 |
| 混合文档 | 按类型自动拆分，分别处理 |

### 导入手写章节（中途接续）

当 AI 已写到第 5 章，你手写了第 6-8 章，需要 AI 从第 9 章继续：

```
AI 写到了第5章，第6到第8章是我自己写的，请接入后从第9章继续
[提供第6-8章]
```

AI 会执行以下步骤：

1. 为你的章节创建章节文件夹
2. 扫描伏笔线索，创建或更新追踪文件
3. 对已有章节做连续性检查
4. 通过 `dp-chapter-summary` 生成章节摘要
5. 生成续写衔接卡，总结当前故事状态
6. 以第 9 章进入正常的 `dp-chapter-draft` 草稿预审

你的手写章节绝不会被修改，只做分析。

### 选择写作模式

在第一章开写前（或任何章节间隙），AI 会询问你选择写作节奏：

| 模式 | 行为 | 适用场景 |
|---|---|---|
| **逐章确认**（默认） | 写作、三阶段审查、你确认、下一章 | 精细控制每一章 |
| **连写 N 章** | 连续完成 N 章（每章仍经审查），统一交付 | 批量推进，事后集中审 |
| **写完一卷暂停** | 写完当前卷的所有章节后暂停 | 按卷管理进度 |
| **全自动** | 写完所有剩余章节，统一交付完整初稿 | 完全信任流程 |

模式切换即时生效，随时可改。

### 中断与恢复

你可以在任何时候中断 AI 的写作：

```
停
暂停
```

继续上次中断的写作：

```
继续写
从第12章开始重写
```

### 中途修改大纲

```
第三章的走向不对，我想改大纲
```

AI 触发 `dp-set-outline` 中的 7 步大纲修订工作流。

### 按需使用质量技能

质量层技能可以随时调用：

```
帮我检查第3章到第7章的连续性
用读者视角测试一下第5章
检查最近三章的AI味
```

| 你的需求 | 调用技能 |
|---|---|
| 连续性检查 | `dp-review-consistency` |
| 读者视角测试 | `dp-review-reader` |
| AI味检测 + 文笔润色 | `dp-review-consistency` |
| 场景导演辅助 | `dp-chapter-direct` |
| 角色风格审查 | `dp-character-style` |

### 作者调优

阅读已完成章节后，如果你想调整后续章节的方向：

```
我看完前五章了，后面节奏太慢，想调整
作者调优
```

AI 加载 `dp-tool-research` 的作者调优模式，引导你确认哪些未起草章节需要调整，收集你的反馈（节奏、伏笔时机、角色戏份、基调转变等），然后将调优意见写入目标章节文件夹的 `tuning.md` 文件。`tuning.md` 在起草时优先级高于大纲默认设定，仅次于铁律。

已完成章节不受影响，走修订流程调整。

### 版本管理

```
提交当前进度
第3章改坏了，回滚到审查通过的版本
```

完整提交规范参见 `dp-tool-version` 技能。

### 切换回编程模式

```
Use skill using-superpowers
```

这会停用所有 `dp-*` 技能，重新进入编程工作流。你的稿件和所有 Dreampowers 制品保留在项目目录中，切换模式不会删除任何内容。

## 制品路径

在项目中使用时，Dreampowers 会创建以下目录结构。`docs/dreampowers/` 目录作为**独立 Git 仓库**管理稿件版本（由 `dp-tool-version` 技能操作）。

**首次使用时初始化**：

```bash
mkdir -p docs/dreampowers/{input,set/{world,concept,character},tracking,timeline,outlines,chapters,release}
cd docs/dreampowers
git init
```

```
docs/dreampowers/                      # 独立 Git 仓库（git init）
├── input/                             # 用户导入的原始数据（临时区）
├── set/
│   ├── world/                         # 世界逻辑关系与背景
│   ├── concept/                       # 概念源文件（一概念一文件）
│   └── character/                     # 角色源文件/目录
├── tracking/
│   ├── overview.md                    # 一句话故事概要
│   ├── iron-rules.md                  # 铁律文件（软链接到各章节文件夹）
│   ├── style.md                       # 写作风格档案（软链接到各章节文件夹）
│   ├── adult.md                       # 成人场景偏好（可选，仅成人章节文件夹有符号链接）
│   ├── thread-NNN-*.md                # 伏笔线索文件（thread- 前缀）
│   └── deferred-threads.md            # 续作保留悬念清单（dp-review-consistency 产出）
├── timeline/
│   ├── timeline.md                    # 初始时间线定义
│   └── summary-NNN.md                 # 每章摘要
├── outlines/
│   ├── outline-*.md
│   └── review-*.md
├── chapters/
│   └── chapter-NNN/
│       ├── spec.md                    # 章节 spec（§1-5 大纲阶段框架 + §6-7 草稿预审写作蓝图）
│       ├── draft.md                   # 草稿（进行中）
│       ├── review.md                  # 章节审查报告
│       ├── tuning.md                  # 作者调优指令（可选，存在时优先级高）
│       ├── adult.md                   # 成人场景偏好（可选，章节级覆盖全局）
│       ├── *.md -> set/concept/*      # 概念符号链接
│       ├── *.md -> set/character/*    # 角色符号链接
│       ├── thread-*.md -> tracking/*  # 伏笔符号链接
│       ├── iron-rules.md -> tracking/iron-rules.md
│       ├── style.md -> tracking/style.md
│       └── summary-*.md -> timeline/summary-*.md  （全部前序章节摘要）
└── release/
    ├── chapter-NNN.md                 # 最终版本
    └── final_report.md                # 全书终检报告（dp-review-final-report 产出）
```

章节文件夹（`docs/dreampowers/chapters/chapter-NNN/`）是完全自包含的写作单元。AI 写作时只读章节文件夹内的文件，不直接访问源目录。只有看不见的信息才能保证不会被使用。

## 与 Superpowers 共存

Dreampowers 与 [Superpowers](https://github.com/obra/superpowers)（编程技能集）完全独立。三层隔离确保互不干扰：

1. **入口切换**：`dp-using-dreampowers` 与 `using-superpowers`
2. **命名空间**：所有 Dreampowers 技能均以 `dp-` 为前缀
3. **目录隔离**：通过符号链接安装，与 Superpowers 源码分离
