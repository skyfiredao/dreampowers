# 读者视角体验测试

本技能是**弹性技能**。四个测试维度的评估标准不可省略，但具体评分的权重可根据章节类型和故事阶段灵活调整。

## 核心定位

这不是作者对自己作品的检查。这是模拟一个真实读者拿起你的章节、从头读到尾时的体验。

类比软件开发：`dp-chapter-draft` 的三阶段审查是代码审查（code review），由开发者视角执行。本技能是用户验收测试（UAT），由消费者视角执行。两者检查的对象相同，但立场完全不同。

作者视角问"这段写得对不对？"，读者视角问"这段我想不想继续读？"。一个章节可以通过所有作者端审查，却让读者在第三页失去兴趣。本技能就是为了捕捉这种落差。

## 与现有技能的关系

三个技能，三个立场，互不替代：

| 技能 | 立场 | 做什么 |
|------|------|--------|
| `dp-chapter-draft` 三阶段审查 | 作者视角 | 质量门禁：情节、揭示、文笔三重检验 |
| `dp-chapter-direct` | 作者视角 | 节奏设计：句长、段落密度、张弛法则 |
| `dp-review-reader`（本技能） | 读者视角 | 消费端体验验证：翻页欲、认知负荷、共情、感受 |

`dp-chapter-direct` 告诉你"动作场景要用短句"。本技能告诉你"读者读到这段时是否真的感到紧张"。一个是处方，一个是疗效检验。

## 输入

| 文件 | 来源 | 说明 |
|------|------|------|
| `docs/dreampowers/chapters/chapter-NNN/draft.md` | 章节文件夹 | 本章草稿（唯一的审阅对象） |
| `docs/dreampowers/chapters/chapter-NNN/summary-*.md` | 章节文件夹内的符号链接 | 前序章节摘要，用于构建冷读者的累积认知 |

**禁止读取当前章节文件夹之外的任何文件。**

## 冷读原则

这是本技能区别于所有其他审查的核心机制。

执行本技能时，你必须切换到"冷读者"人格。冷读者只知道已经写出来的内容，不知道任何幕后信息：

- **不参考大纲**。不知道后续剧情走向，不知道后面的反转
- **不参考世界观设定完整版**。只知道已在正文中揭示的世界观设定
- **不参考角色设定完整版**。只知道已通过行动和对话展现的角色信息
- **不参考揭示时间表**。不知道作者计划在第几章揭示什么
- **不参考伏笔场记**（`docs/dreampowers/tracking/thread-*.md`）。不知道哪些是伏笔，哪些是闲笔

仅凭已写章节的累积信息，逐段判断。

如果某个角色的动机只写在角色设定里但从未在正文中展示，冷读者会标记"我不理解这个角色为什么这么做"。如果某个设定在设定文档中详细定义但正文只提了一句，冷读者会标记"这个设定让我困惑"。这正是本技能要捕捉的问题。

## 测试维度

四个维度，每个独立评分，1-5 分制。

### 维度一：翻页欲测试

核心问题：**读者读完这一段/这一章后，是否想继续读下去？**

检查要素：
1. **章尾钩子**：本章最后一段是否留下悬念、承诺、或未解决的紧张？章节结束在"事情都处理完了"的状态 = 翻页欲归零
2. **中段悬念**：章节中间是否有持续的未回答问题在牵引阅读？还是只在开头和结尾有钩子，中间一片平坦
3. **信息承诺**：是否让读者感觉到"继续读就能知道 X"？X 不必是大悬念，可以是角色的一个小决定、一段对话的结果
4. **未解答问题计数**：到本章结尾，读者心里有几个"还没搞清楚"的问题？理想 2-4 个。0 个 = 没动力。>6 个 = 混乱

评分标准：

| 分数 | 含义 |
|------|------|
| 5 | 读者会说"不行我必须再看一章" |
| 4 | 读者想继续，但可以放下书去做别的事 |
| 3 | 读者没有特别想停也没有特别想继续 |
| 2 | 读者可能在这里放下书，需要外部动力才会继续 |
| 1 | 读者大概率弃书 |

**任何章节评分 ≤2 必须标记为高优先修改项。**

### 维度二：认知负荷测试

核心问题：**读者是否感到困惑或被信息淹没？**

检查要素：
1. **新概念密度**：本章引入了几个读者此前未接触的名词、设定、规则？统计时只看正文累积，不看世界观设定
2. **无上下文元素**：有没有突然出现的名字、地点、术语，之前从未提及也没有即时解释？
3. **信息密度分布**：每个场景承载了多少新信息？是否存在某个场景塞了太多新概念？
4. **预设知识缺口**：作者是否假设读者知道某些从未在正文中交代的事？

量化红线：
- 首 500 字内出现 >3 个未经铺垫的新元素 = 认知过载
- 单个场景内引入 >2 个全新概念 = 危险区
- 连续三段无任何熟悉锚点（已知角色、地点、情感）= 读者漂流

评分标准：

| 分数 | 含义 |
|------|------|
| 5 | 认知负荷极高，读者需要反复翻阅才能理解 |
| 4 | 明显吃力，部分段落需要重读 |
| 3 | 有一两处需要短暂停顿消化 |
| 2 | 轻微负荷，阅读顺畅 |
| 1 | 零负荷，所有信息自然流入 |

注意：认知负荷评分**越低越好**。1 分是最佳状态。

### 维度三：共情验证

核心问题：**读者能否感受到角色感受到的东西？读者是否在乎角色的遭遇？**

检查方法：在每个关键情节转折点执行"为什么我应该在乎？"测试：
1. 暂停阅读，仅凭已读内容用一句话概括：这个角色此刻想要什么？
2. 概括清晰具体（"李明想保住女儿的手术费"）= 共情通道打开
3. 概括模糊空泛（"李明想解决问题"）= 共情通道未建立
4. 无法概括（"我不知道这个角色为什么在做这件事"）= 共情失败

检查要素：
1. **动机透明度**：角色行动的原因是否可以从已有文本推断？不需要直说，但必须有足够线索
2. **情感锚定**：读者是否有过与角色共享情感的时刻？至少每章一次
3. **角色目标可见性**：角色在追求什么？读者能看到吗？
4. **代价感知**：角色如果失败会怎样？代价越具体，共情越强

评分标准：

| 分数 | 含义 |
|------|------|
| 5 | 读者为角色捏一把汗，情感完全同步 |
| 4 | 读者理解并关心角色处境 |
| 3 | 读者理解角色但情感距离较远 |
| 2 | 读者对角色遭遇基本无感 |
| 1 | 读者不理解角色在干什么或者为什么 |

### 维度四：节奏感受测试

核心问题：**读者主观上觉得这一章的节奏舒服吗？**

这与 `dp-chapter-direct` 的客观节奏分析不同。那个技能量化句长、段落密度、张弛比例。本维度只关心结果：读者读起来的感受。

两个检验句：

1. **"我想跳过这一段吗？"**：如果是，该段落拖沓。可能是信息重复、情感原地踏步、描写过度、或缺乏细节张力
2. **"等等，刚才发生了什么？"**：如果是，该段落过急。可能是跳过了必要的过渡、省略了情感反应、或信息压缩过度

检查要素：
1. 标记所有触发"想跳过"感觉的段落，注明拖沓原因
2. 标记所有触发"等等什么"感觉的段落，注明过急原因
3. 评估整体节奏是否匹配章节类型
4. 注意节奏单调性：是否全章一个速度走到底？

评分标准：

| 分数 | 含义 |
|------|------|
| 5 | 完全沉浸，没有注意到"节奏"这个东西的存在 |
| 4 | 整体顺畅，有个别段落微微出戏 |
| 3 | 有明显的拖沓段或过急段，但整体可接受 |
| 2 | 节奏频繁出问题，多次打断阅读沉浸 |
| 1 | 无法持续阅读，反复出戏 |

## 执行流程

调用本技能后，严格按以下五步执行。

### 第一步：冷读模式

切换到冷读者人格。放下所有创作者知识，前序章节信息仅从章节文件夹内的 `summary-*.md` 符号链接获取。从第一个字开始逐段阅读，记录即时感受，不做回溯修正。第一反应就是最真实的读者反应。

### 第二步：逐维度评分

读完全章后，回到四个维度，逐一评分。每个维度给出 1-5 分，并写一段 2-3 句话的评分依据。

### 第三步：问题标记

回到正文，标记具体段落或句子。格式：

```
[第 X 段] 翻页欲问题：此处连续三段描写环境，无细节张力，读者可能脱离
[第 Y 段] 认知负荷问题："冥火" "灵脉" "吞天诀"三个新术语在同一段出现，无解释
[第 Z 段] 共情问题：角色突然暴怒，但前文未建立情感积累，读者会觉得突兀
```

### 第四步：修改建议

针对每个标记的问题，给出方向性建议。不给具体改写方案，只指明方向。正确示例："第 5 段需要在描写中插入角色的内心反应，让环境描写承载情感而非纯粹展示"。错误示例：直接改写该段落。

### 第五步：输出报告

按以下格式生成完整报告：

```
## 读者体验报告：第 X 章

### 总评
翻页欲: X/5 | 认知负荷: X/5 (越低越好) | 共情: X/5 | 节奏: X/5

### 翻页欲
[评分依据]
[具体发现]

### 认知负荷
[评分依据]
[具体发现]

### 共情验证
[评分依据]
[具体发现]

### 节奏感受
[评分依据]
[具体发现]

### 优先修改建议
1. [最高优先级]
2. [次优先级]
3. ...
```

## 触发时机

1. **外部审阅闭环（必选）**：章节通过 `dp-chapter-draft` 的三阶段审查后，自动进入外部审阅闭环。本技能在闭环中**先于** `dp-review-consistency` 执行，获取读者视角的问题标记。闭环最多循环 3 次（详见 `dp-chapter-draft` 的外部审阅闭环章节）
2. **用户明确要求**：用户说"从读者角度看看这章"、"这章读起来怎么样"、"帮我做读者体验测试"
3. **连续写作中的定期执行**：连写N章 / 全自动模式中，每 5 章强制执行一次，防止作者视角过深

## 特殊场景处理

### 连载首章

翻页欲权重加倍。读者在第一章决定是否追更，这是生死线。特殊检查项：
- 首 300 字内是否建立了至少一个悬念或承诺？
- 首 500 字内是否出现了可以产生共情的角色时刻？
- 章末是否有足够强的钩子让读者点击"下一章"？

翻页欲评分 ≤3，首章不合格。不是建议修改，是必须修改。

### 高潮章节

节奏感受权重加倍。高潮章节如果节奏拉胯，读者体验直接崩塌。特殊检查项：
- 是否存在"想跳过"的段落？高潮章节中任何让人想跳过的内容都是严重问题
- 情感高峰是否真的让人感到冲击？还是平淡地滑过去了？
- 节奏是否有层次？纯粹的高速推进也会让人疲劳

### 过渡章节

认知负荷容忍度降低。过渡章节是读者耐心最薄弱的时候。特殊检查项：
- 不要在过渡章节塞入大量新设定，读者此时期待喘息
- 认知负荷评分 ≥4（高负荷）时，过渡章节直接不合格
- 翻页欲下限 3 分，低于此值说明过渡章节变成了填充章节

## 注意事项

- 所有评判必须基于冷读原则。如果你在引用大纲或世界观设定中的信息为某段辩护，说明已脱离冷读者视角，需要重新校准
- 评分要诚实。4 分和 5 分差距很大，不要轻易给 5 分。3 分是真正的中间值，意味着"还行但没什么印象"
- 修改建议只给方向。本技能的产出是诊断报告，不是修改稿。具体改写交给用户决定，或交给 `dp-review-consistency` 执行
- 四个维度中两个或以上评分 ≤2，建议整体性修改，而非逐点修补

## 与其他技能的交互

| 关系 | 技能 | 说明 |
|------|------|------|
| 被调用 | `dp-chapter-draft` | 三阶段审查通过后，在外部审阅闭环中被调用，先于 dp-review-consistency 执行 |
| 上游 | `dp-chapter-summary` | 使用前序章节摘要构建冷读者的累积认知 |
| 协作 | `dp-review-consistency` | 同在外部审阅闭环中，本技能先执行，dp-review-consistency 后执行。两份报告的问题汇总后统一修改 |

## 反模式

- ❌ **引用大纲或世界观设定为文本辩护** — 冷读者看不到那些信息，引用即脱离视角
- ❌ **给出具体改写方案** — 本技能只诊断，不改写。修改交给用户或 dp-review-consistency
- ❌ **所有维度都给 4-5 分** — 评分通胀让诊断失去意义。3 分是"还行但没什么印象"
- ❌ **跳过特殊场景权重调整** — 首章、高潮章、过渡章各有权重规则，不可一视同仁
- ❌ **只看单章不看累积** — 认知负荷和翻页欲需要对照前序章节的累积状态
